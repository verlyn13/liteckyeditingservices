# CMS Quick Reference

A quick guide for editing content on the Litecky Editing Services website.

---

## Logging In

1. Go to: **https://www.liteckyeditingservices.com/admin/**
2. Click **"Login with GitHub"**
3. Authorize if prompted (first time only)

---

## Making Changes

1. Click the collection you want to edit (e.g., **Client Testimonials**)
2. Click an existing item to edit, or **"New"** to create
3. Make your changes in the form
4. Click **Save** (creates a draft)
5. Change status from "Draft" to **"Ready"**
6. Click **Publish**

---

## Important Notes

- Changes take **~2 minutes** to appear on the live site
- Check build status via the link in the yellow banner at the top
- If something looks wrong, wait 2 minutes and refresh the page
- All changes are saved to GitHub and can be undone if needed

## Collection Guide

### Page Settings

Edit the main content on Home and About pages:

- **Home Page**: Hero headline, subtitle, buttons, process steps, featured testimonial
- **About Page**: Headline, intro text, founder info, expertise list

### Client Testimonials

Add or edit client feedback:

- **Client Name**: Use first name and last initial (e.g., "Sarah M.")
- **Role**: Their title (e.g., "PhD Candidate")
- **Institution**: University or organization
- **Rating**: 1-5 stars
- **Feature on Homepage**: Toggle ON to highlight this testimonial

### Services

Manage your service offerings:

- **Service Name**: Clear, descriptive title
- **Short Description**: One sentence for service cards
- **Price**: Format as you prefer (e.g., "$0.02/word" or "Starting at $150")
- **Features**: Bullet points of what's included
- **Display Order**: Lower numbers appear first

### FAQ

Add frequently asked questions:

- **Question**: Write as a client would ask it
- **Short Answer**: 1-2 sentences (shown collapsed)
- **Category**: Groups related questions together
- **Detailed Answer**: Full explanation with formatting

---

## Tips for Great Content

### Writing Headlines
- Keep headlines under 10 words
- Focus on benefits to the reader
- Use action words when appropriate

### Testimonials
- 2-4 sentences is the ideal length
- Use the client's actual words
- Include specific details when possible

### Descriptions
- Lead with the most important information
- Use simple, clear language
- Break long text into short paragraphs

## Troubleshooting

### Changes not appearing?
1. Wait 2 minutes after publishing
2. Hard refresh the page (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows)
3. Check the build status link in the yellow banner

### Can't log in?
1. Make sure you're using the correct GitHub account
2. Try clearing your browser cache
3. Contact Jeffrey if issues persist

### Made a mistake?
- All changes are tracked in GitHub
- Jeffrey can revert any changes if needed
- Don't worry - nothing is permanently lost!
